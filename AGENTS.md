# AGENTS.md

本文件位于项目根目录，作用域覆盖整个仓库。除非有更高优先级指令，后续所有改动都必须遵循以下约束。

## 项目边界
- 项目类型：**原生微信小程序**。
- 不接后端服务：**不新增、不开发、也不依赖自建或第三方业务后端接口**。
- 不做大数据能力：**不引入数据仓库、实时计算、埋点平台、用户画像等大数据方案**。
- 数据流转方式：仅允许 **`storage + mock`**。
  - 持久化使用小程序本地存储（如 `wx.setStorageSync` / `wx.getStorageSync`）。
  - 业务数据来源使用本地 mock 数据（如 `mock/` 目录、静态 JSON、内置模拟模块）。
  - 禁止引入远程 API 请求作为业务主数据来源。

## 代码风格规范
- 使用清晰、可维护、与现有项目一致的命名与结构。
- 保持函数职责单一，避免过长函数和重复逻辑。
- 公共能力优先沉淀到复用模块，避免在页面中复制粘贴。
- 变更应最小化：只修改与需求直接相关的文件。
- 新增/修改代码时补充必要注释，重点说明“为什么”，避免无意义注释。

## 目录与文件组织规范
- 页面相关文件按小程序原生约定同目录组织（如 `index.js / index.wxml / index.wxss / index.json`）。
- mock 数据集中管理，建议放在 `mock/` 或项目既有 mock 目录下。
- 工具函数与通用模块放在统一的 `utils/`（或项目既有公共目录）下。
- 静态资源放在统一资源目录（如 `assets/` 或项目既有目录），避免散落。
- 非必要不新增根目录层级；如需新增目录，需与现有结构风格保持一致。

## 执行要求
- 从本文件创建后开始，默认所有后续改动均需显式遵守本约束。
- 如出现冲突，以系统/开发者/用户的即时指令为更高优先级。
